Stripe Secure Payment Module-Anweisungen fur Zen-Cart-Version 1.5.7d

Einfuhrung

  1. Dieses Modul wird fur die sichere Zahlung mit Stripe verwendet. Sie mussen sich bei der Firma Stripe anmelden und einen Vertrag abschliesen.
  2. Stripe unterstutzt die Abwicklung von Zahlungen in uber 135 Wahrungen und das Modul unterstutzt auch uber 135 Wahrungen.
  3. Sie sollten diese Anweisungen vor der Installation, nach der Installation und wahrend des Betriebs befolgen.
  • Vor der Installation: Sichern Sie alle Datenbanken, Ordner und Dateien. Sichern Sie insbesondere uberschriebene Dateien. Die Installation erfolgt auf eigene Verantwortung.
  • Nach der Installation: Uberprufen Sie den Betrieb im Testmodus. Prufen Sie auf der Stripe-Dashboard-Seite , ob der Zahlungsbetrag korrekt ist . Vergleichen Sie mit der Gesamtsumme der Bestellung auf der Zen-Warenkorb-Administrationsseite Kunden => Bestellungen. Stellen Sie sicher, dass beide gleich sind.
  • Betrieb: Der Testmodus ist der Standardwert, wenn dieses Modul installiert ist. Vergessen Sie nach dem Test nicht, den Testmodus auf ?false“ zu setzen. Geben Sie den von Stripe bereitgestellten geheimen Schlussel nicht preis .
  • WICHTIG: Nachdem Sie den API-Schlussel geandert haben, mussen Sie die folgenden Schritte ausfuhren.
    Gehen Sie zu Admin => Tools => Seite SQL-Patches installieren. Laden Sie die Datei erase_stripe_recordes.sql hoch oder fugen Sie den PHP-Code ein TRUNCATE ` stripe `; in das Textfeld "Geben Sie die auszufuhrende Abfrage ein:" ein. und drucken Sie die Schaltflache "Senden".
  • Wir übernehmen keine Haftung und Verantwortung für Schaden, Verluste und Kosten, die durch die Verwendung von dieses Modul verursacht werden, daher mussen Sie das Modul im Testmodus uberprufen und bestatigen, dass es auch nach dem Testen normal funktioniert.

Anforderungen

  1. Zen-Cart-Version 1.5.7d
  2. SSL (SSL ist erforderlich. Wenn Sie ohne SSL installieren, tritt ein Fehler auf, nachdem Sie auf die Schaltflache "Bestellung bestatigen" geklickt haben.)
  3. PHP 5.6 bis 8.0

Installation

  1. Entpacken Sie die heruntergeladene Datei.
  2. Andern Sie den Namen des Vorlagenordners von YOUR_TEMPLATE in Ihren Vorlagennamen.
    • Benennen Sie den Ordner YOUR_TEMPLATE in Stripe_module_zc157d/www/includes/templates in Ihren Vorlagennamen um.
  3. Die folgenden 5 Dateien werden uberschrieben. Wenn eine dieser Dateien geandert wird, fuhren Sie sie zusammen, uberschreiben Sie sie nicht.
    • Stripe_module_zc157d\www\includes\templates\YOUR_TEMPLATE\templates\tpl_account_default.php
    • Stripe_module_zc157d\www\includes\templates\YOUR_TEMPLATE\templates\tpl_checkout_confirmation_default.php
    • Stripe_module_zc157d\www\includes\templates\YOUR_TEMPLATE\templates\tpl_checkout_payment_default.php
    • Stripe_module_zc157d\www\includes\languages\english\lang.account.php
    • Stripe_module_zc157d\www\includes\database_tables.php
    • Stripe_module_zc157d\www\includes\init_includes\init_header.php
  4. Laden Sie alle Dateien hoch
    • Fugen Sie den Ordner Stripe_module_zc157d =>www=>includes in den Ordner include des Zen-Cart ein.
  5. Die Spalte "Stripe-Zahlung: Kreditkarte" wird auf der Seite admin=>Module=>Paymet hinzugefugt. und wahlen Sie es aus.
  6. Nachdem Sie auf die Schaltflache ?+ Modul installieren“ geklickt haben, wird das Stripe-Zahlungsmodul installiert und die folgenden Elemente werden angezeigt.
    • Sicheres Stripe-Zahlungsmodul aktivieren: True/False Standard: True
    • Testmodus Sicheres Stripe-Zahlungsmodul: True/False Standard: True
    • Zahlungszone: Wenn eine Zone ausgewahlt ist, aktivieren Sie diese Zahlungsmethode nur fur diese Zone. Standard:leer
    • Bestellstatus festlegen: Setzt den Status von Bestellungen, die mit diesem Zahlungsmodul getatigt wurden, auf diesen Wert Standard: 0
    • Veroffentlichbarer API-Schlussel: Geben Sie den veroffentlichbaren API-Schlussel ein, der von Stripe bereitgestellt wird
    • API-Geheimschlussel: Geben Sie den von Stripe bereitgestellten API-Geheimschlussel ein
    • Testmodus ? API-veroffentlichbarer Schlussel: Geben Sie den von Stripe bereitgestellten API-veroffentlichbaren Testschlussel ein
    • Testmodus - Geheimer API-Schlussel: Geben Sie den von Stripe bereitgestellten geheimen API-Testschlussel ein

Testmodus - Stripeenzahlungsmodus (Sie mussen das Modul vor dem Betrieb uberprufen.)　

  1. Admin-Seite (Testmodus)
    • Geben Sie API Publishable key , API Secret key , Test Mode API Publishable Test Key und Test Mode - API Secret Test Key ein. und Drucken Sie die Schaltflache Aktualisieren.
  2. Titelseite speichern (Testmodus)
    • Legen Sie einige Produkte in den Warenkorb. und gehen Sie zur Check-out-Seite.
    • Nach Auswahl des Stripe-Zahlungsformulars: Kreditkarte wird das Stripe-Kreditkartenformular wie folgt angezeigt. Wenn das Kreditzahlungsformular nicht angezeigt wird, haben Sie moglicherweise den falschen API-Schlussel eingegeben.
    • Geben Sie die Kreditkartennummer des Testvisums ein 4242 4242 4242 4242 、 Ablaufdatum ist nach dem nachsten Monat , Geben Sie eine zufallige 3-stellige Zahl in den CVC ein. Und drucken Sie die Schaltflache "Bestellung bestatigen".
    • Gehen Sie zum Stripe-Dashboard. Klicken Sie auf die Schaltflache ?Zahlungen und erfolgreich“. Uberprufen Sie die Informationen zum Gesamtbetrag, zum Kundennamen und zur E-Mail-Adresse.

      　HINWEIS: Das System von Stripe multipliziert Wahrungen mit 2 Dezimalstellen, wie z. B. USD und EUR, mit 100, bevor sie an die Server von Stripe gesendet werden. Der Stripe-Server konvertiert die empfangene Zahl in 1/100 und verwaltet sie. Japanische Yen haben kein Dezimalkomma, daher wird die Zahl unverandert an Stripe-Server gesendet. Dieses Modul konvertiert automatisch fur jede Wahrung, aber testen Sie es mit allen Wahrungen, die Sie verwenden, bevor Sie es verwenden. Wir übernehmen keine Haftung und Verantwortung für Schäden, Verluste und Kosten, die durch die Verwendung von dieses Modul verursacht werden.

      So loschen Sie die Kreditkarteninformationen des Kunden von der Storefront-Seite.
      　Die Kreditkarteninformationen des Kunden werden verschlusselt und in der Datenbank registriert, konnen aber nach Ermessen des Kunden geloscht werden.
      　Der Kunde kann sie mit der Stripe-Schaltflache zum Loschen von Kreditkarteninformationen auf der Seite ?Mein Konto“ loschen. Nach dem Loschen verschwindet diese Anzeige.

      Betrieb (nach dem Test)

      1. Gehen Sie zu Admin => Module => Zahlungsseite. Wahlen Sie Stripe Payment : Credit Card und klicken Sie auf die Schaltflache Bearbeiten.
      2. Schalten Sie das Optionsfeld Testmodus Stripe Secure Payment Module von True auf False um. und drucken Sie die Schaltflache Aktualisieren.
      3. Stellen Sie sicher, dass im Feld Stripe Payment : Credit Card nichts rot angezeigt wird.
      4. Stellen Sie sicher, dass der rote "STRIPE IS IN TESTING MODE" nicht oben auf den Storefront-Seiten angezeigt wird.

      WICHTIG: Nachdem Sie den API-Schlussel geandert haben, mussen Sie die folgenden Schritte ausfuhren.
      Gehen Sie zu Admin => Tools => Seite SQL-Patches installieren. Laden Sie die Datei erase_stripe_recordes.sql hoch oder fugen Sie den PHP-Code ein TRUNCATE ` stripe `; in das Textfeld "Geben Sie die auszufuhrende Abfrage ein:" ein. und drucken Sie die Schaltflache "Senden".
      HINWEIS: Nach dem Andern des API-Schlussels die Bedeutung der Ausfuhrung des obigen SQL. Der API-Schlussel und die Stripe-Kunden-ID hangen zusammen, wenn also der API-Schlussel geandert wird, wird die falsche Stripe-Kunden-ID an den Stripe-Server gesendet und das Stripe-Zahlungsformular verschwindet. Alle Stripe-Kunden-IDs werden durch das oben beschriebene Verfahren geloscht.

      HINWEIS: Wenn Sie das Stripe-Dashboard => Zahlungen => Alle Seite offnen, finden Sie viele unvollstandige Zahlungen. Weil der Stripe-Server jedes Mal pi_xxxxxxxxxxxxxxx erstellt, wenn die Check-out-Bestatigungsseite des Zencart-Warenkorbs angezeigt wird. und der Stripe-Server erstellt zwei pi_xxxxxxxxxxxxxxx fur jede Zahlung. Die Ursache liegt in der Spezifikation von Stripe-Server und Zencart.

      Deinstallation

      1. Gehen Sie zur Seite Admin=>Module=>Zahlung. Wahlen Sie "Stripe-Zahlung: Kreditkarte". und drucken Sie die Deinstallationstaste.
      2. Gehen Sie zu Admin => Tools => Seite SQL-Patches installieren. Laden Sie die Datei uninstall_stripe.sql hoch oder fugen Sie den PHP-Code ein DROP TABLE IF EXISTS `stripe`; in das Textfeld "Geben Sie die auszufuhrende Abfrage ein:" ein. und drucken Sie die Schaltflache "Senden".
